Validates the joining on condition.

The on parameter in the joining conditions needs to have a proper syntax in order to validate successfully. You can provide multiple conditions, and they all need to be valid.

A valid joining condition uses the equality expression between two special fields called left (the original query context) and right (the joined query context). You can also specify just the field name if both queries include it. 


data json: "..."
| join [...] on: id // if the "id" is in both queries, the condition is OK
| join [...] on: left[dt.entity.host] == right[id] // if the fields are different, you need to use the "left" and "right" fields
// you can also provide multiple conditions using a valid syntax for each of them
| join [...] on: {
    left[service.id] == right[id],
    left[cloudType] == right[cloud],
    dt.entity.host
}
