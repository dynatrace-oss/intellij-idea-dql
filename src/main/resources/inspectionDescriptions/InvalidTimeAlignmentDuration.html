Checks if the expression used in the time alignment is a valid target.

This inspection reports an error if:

  1. the left side of the expression uses either a disallowed type of the expression
  2. the value of that expression is not of type that could be used in the time alignment expressions (like timestamps or timeframes).

fetch logs,
    from: now()@d, // now() returns a timestamp, so it is a valid operand
    to: (test desc)@w, // sorting expression is not a valid operand
    timeframe: 55@w // positive long "55" is not a valid operand

See all supported operation types in the official Dynatrace documentation.
