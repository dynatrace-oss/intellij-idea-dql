Checks usage of nested negative values.

Nested negative values are much harder to understand, whether the value is positive or negative. The inspection reports a notice that the value could be simplified by removing unnecessary - signs. Although the expression is perfectly valid on DQL, it still useful for code style quality remove unnecessary negative values. 


data record()
  | fields {
      field = ------55,  // is the same as "field = 55"
      field2 = -------55 // is the same as "field2 = -55"
  }

After applying the quick-fix, the negative value will be simplified.
