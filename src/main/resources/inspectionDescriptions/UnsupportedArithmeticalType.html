Checks if both sides of an arithmetical operation are valid.

This inspection focuses on the validity of types returned by expressions used in the operation. The information shown to the user will compare all possible types for the left, and the right side, with values returned by the expression. To be a deemed valid expression, both sides must be supported data types. See all supported operation types in the official Dynatrace documentation. 


data json: "..."
| fields field = 22 * "string", // string is not a valid arithmetical target when multiplying numbers
    field2 = now() / 5 // timestamp cannot be used as a left side for the divide operation
